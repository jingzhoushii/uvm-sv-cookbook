# 📊 考核评估标准

## 概述

本目录包含所有考核相关的评估标准和样例。

---

## 考核结构

| 考核 | 占比 | 时间 |
|------|------|------|
| 周考 1 (SV) | 15% | 60 分钟 |
| 周考 2 (UVM) | 15% | 60 分钟 |
| 周考 3 (RAL) | 15% | 60 分钟 |
| 项目 | 40% | 2 周 |
| 笔试 | 15% | 120 分钟 |

**总分 60% 及格**

## 周考 1：SystemVerilog（60 分钟）

### 题型分布
| 类型 | 题数 | 分值 |
|------|------|------|
| 选择题 | 20 | 40 分 |
| 填空题 | 10 | 20 分 |
| 编程题 | 2 | 40 分 |

### 样题

#### 选择题（2 分/题）
1. 哪个是二值逻辑类型？
   A. logic  B. wire  C. reg  D. bit

2. 哪个不是四值逻辑？
   A. wire  B. reg  C. int  D. logic

#### 填空题（2 分/题）
1. `bit [7:0] a = 'hFF;` 中 a 的十进制值是 _______
2. `int i = -5;` 中 i 的二进制补码是 _______

#### 编程题（20 分/题）
1. 实现带约束的 Transaction（addr < 100, data > 0）
2. 实现 Mailbox 生产者-消费者

## 项目评估（40%）

### 评估维度

| 维度 | 分值 | 说明 |
|------|------|------|
| 代码质量 | 10% | 风格、规范、可读 |
| 功能完整 | 15% | 功能实现程度 |
| 测试覆盖 | 10% | 覆盖率、测试用例 |
| 文档 | 5% | README、注释 |

### 扣分项

| 扣分 | 原因 |
|------|------|
| -5% | 代码无法编译 |
| -5% | 缺少文档 |
| -10% | 功能不正确 |
| -5% | 命名不规范 |

## 通过标准

### 分数计算
```
总分 = 周考1×15% + 周考2×15% + 周考3×15% + 项目×40% + 笔试×15%
```

### 及格线
- **60% 及格**
- **70% 良好**
- **85% 优秀**

### 补考
- 不及格（<60%）可申请补考
- 补考分数上限 60%

## 证书

### 证书等级
| 等级 | 分数 | 要求 |
|------|------|------|
| A | ≥85% | 所有考核 ≥75% |
| B | ≥70% | 所有考核 ≥60% |
| C | ≥60% | - |
